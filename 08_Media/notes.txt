HTML MEDIA :-
==============
-> Overview:
 - HTML supports various media elements like audio, video, and embedded content to enhance user engagement.
 - Commonly used tags: <audio>, <video>, <source>, <embed>, <iframe>.

-> <audio> Tag:
 - Used to embed sound/music/audio files.
 - Attributes: controls (show play UI), autoplay, loop, muted.

 - Syntax
   <audio controls>
     <source src="song.mp3" type="audio/mpeg">
     <source src="song.ogg" type="audio/ogg">
     Your browser does not support the audio element.
   </audio>

-> <video> Tag:
 - Used to embed video content.
 - Attributes: controls, width, height, autoplay, loop, muted, poster.

 - Syntax
   <video width="320" height="240" controls>
     <source src="movie.mp4" type="video/mp4">
     <source src="movie.webm" type="video/webm">
     Your browser does not support the video tag.
   </video>

-> <embed> Tag:
 - Embeds external media like PDFs, Flash, animations, etc.
 - Self-closing tag.

 - Syntax
   <embed src="file.pdf" width="500" height="400">

-> <iframe> Tag:
 - Embeds another HTML page or content (e.g., YouTube video) inside the page.

 - Syntax
   <iframe src="https://www.youtube.com/embed/videoID" width="560" height="315"></iframe>

-> Best Practices:
 - Always provide controls for better UX.
 - Use multiple <source> formats for wider browser support.
 - Avoid autoplay with sound (bad user experience).
 - Use proper width and height to preserve layout.

-> Expert Tips:
 - Combine <audio> and <video> with the preload attribute (auto | metadata | none) for performance.
 - Use CSS to make iframes responsive.
 - Optimize media file size before embedding to improve load times.
